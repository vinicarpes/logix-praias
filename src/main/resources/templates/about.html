Informações adicionais

O sistema Logix Praias surgiu para suprir uma demanda logística do Corpo de Bombeiros Militar de Santa Catarina (CBMSC), mais especificamente do Grupamento de Busca e Salvamento (GBS). Este, é responsável pelo gerenciamento das praias de Florianópolis durante todo o ano, tanto em questões de resgate, abastecimento de materiais, etc.

A proposta do projeto é fornecer um controle logístico eficaz, transparente e disponível aos envolvidos na Operação Veraneio na região da ilha de Santa Catarina.

Dito isso, a solução proposta inclui:

  • Cadastro de materiais por categoria (APH, Limpeza...)
  • Registro de recebimento e retirada do estoque
  • Notificação por retirada e/ou recebimento de materiais
  • Relatórios de estoque

Para o desenvolvimento, foi utilizado a linguagem de programação Java e o ecossistema spring para o backend, enquanto o frontend foi construído com Javascript, HTML, CSS e Bootstrap.

Pensando em melhorias para a aplicação, visando a escalabilidade, disponibilidade e segurança, os próximos passos a serem tomados incluem:

  • Desacoplar as stacks (dividir backend e frontend)
  • Implementar autenticação de acesso por usuário
  • Utilização de frameworks escaláveis no frontend (React, Vue)
  • Integração com sistemas do CBMSC

É importante ressaltar que esse projeto foi desenvolvido por livre e espontânea vontade e sem custos à corporação.

O código fonte e documentação do projeto pode ser consultado no repositório do github.
